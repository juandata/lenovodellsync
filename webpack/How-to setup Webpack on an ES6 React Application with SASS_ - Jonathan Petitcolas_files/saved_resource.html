                    pfaz 3 years ago

                    Good article. Couple of small points of feedback :

                    - Rather than having to keep referencing webpack in node_modules every time you want to run, it's common to install it globally or alternatively add a relative path to the bin folder in your path.

                    - Another approach I prefer is to put the commands into your package.json, for instance we have "npm run build|serve|dev" which all launch webpack/webpack-dev-server passing the necessary args.

                    - take a look at the webpack-dev-server --inline option. This inlines the webpack-dev-server code so you don't need to change your entry point/config at all and in the case above you don't need the getEntrySources function (although there are other good reasons you might want to have separate webdev dev & production configs). i don't see --inline mentioned in many articles with most choosing to add webpack-dev-server/client?http://localhost:8080 but there's no advantage to doing this afaics.
